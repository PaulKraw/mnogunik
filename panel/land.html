01
📋
Этап 11 · Подготовка
Прайс-лист и каталог
Собираем все товары и услуги в единую таблицу-основу
▼
Суть этапа
Всё начинается с данных о товарах или услугах. Нужно собрать полную таблицу, где каждая строчка — это один конкретный товар или услуга со всеми характеристиками. Это будет «база» — из неё будут созданы все объявления.

Если товар продаётся в разных конфигурациях (например, хлеб ржаной 400 г и 700 г, или компьютер с SSD 256 ГБ и 512 ГБ) — используется инструмент pricecraft, который автоматически «разворачивает» все комбинации в отдельные строки. Каждая строка = один SKU (отдельная позиция).
Кто что делает
👤 Человек (вы / клиент)
  • Предоставляет прайс-лист (Excel, Google Sheets или просто список)
  • Указывает города присутствия
  • Указывает категорию товаров/услуг
  • Согласовывает базовые цены
🤖 Специалист + скрипты
  • Приводит данные к нужному формату
  • Запускает pricecraft для разворачивания конфигураций
  • Сохраняет результат как catalog.csv в базу данных
  • Проверяет полноту и корректность данных
Поток данных
📥 Прайс от клиента (Excel/Sheets) → 🔧 pricecraft + Python → 📄 catalog.csv (база SKU)
Пример
Услуга «Испечение хлеба»: одна строка — название, описание, город, цена от 500₽/кг.
Товар «Хлеб и пироги»: хлеб ржаной 400г — строка 1, хлеб ржаной 700г — строка 2, пирог с яблоком 500г — строка 3, пирог с вишней 500г — строка 4. Итого 4 SKU из одного прайса.
Что нужно от вас
✅Список товаров/услуг с ценами в любом виде (Excel, Google Таблицы, Word, даже фото прайса)
✅Список городов, где хотите размещаться
✅Примерное описание целевой аудитории (кто ваш покупатель)
02
💡
Этап 2 · Контент
Маркетинговые гипотезы
Создаём несколько вариантов объявлений для тестирования
▼
Суть этапа
Одного текста недостаточно — непонятно, что сработает лучше. Поэтому создаётся несколько вариантов (гипотез): разные заголовки, описания, картинки. Потом их публикуют и смотрят, какой приносит больше кликов и звонков при меньших затратах.

Тексты генерирует Claude API (нейросеть) — она получает характеристики товара и шаблон, и создаёт уникальный продающий текст для каждого объявления. Картинки формируются автоматически из шаблона: берётся заготовка (фон, логотип) и на неё подставляется цена, скидка или ключевая характеристика.

Кроме текста и картинки, у каждого объявления должны быть уникальные поля — Авито проверяет, не является ли новое объявление копией существующего. Поэтому скрипт генерирует случайные значения: например, артикул «АВТ-7842», вес «0.41 кг», высота «18.2 см».
Кто что делает
👤 Специалист
  • Формулирует 2–4 гипотезы: «акцент на скорость», «акцент на натуральность», «акцент на цену»
  • Задаёт шаблоны промптов для Claude API
  • Проверяет и при необходимости редактирует сгенерированные тексты
  • Готовит шаблоны картинок (один раз)
🤖 Claude API + Python
  • Генерирует уникальный текст для каждого SKU × каждой гипотезы
  • Создаёт картинки через Pillow: подставляет цену, текст, характеристики
  • Генерирует уникальные поля через Python random
  • Сохраняет все варианты в variants.csv
Поток данных
📄 catalog.csv + 📝 Шаблоны текстов → 🤖 Claude API → 📝 Уникальные тексты
🖼️ Шаблон картинки → 🖼️ Pillow (PIL) → 🖼️ Готовые картинки
🎲 Python random → 🔢 Уникальные поля (артикул, вес, высота)
📄 variants.csv (все варианты для публикации)
Пример гипотез
Услуга:
Гипотеза А: «Свежий хлеб на заказ — выпекаем за 3 часа, доставим горячим»
Гипотеза Б: «Домашняя выпечка без консервантов — хлеб и пироги по бабушкиным рецептам»

Товар:
Гипотеза А: «Ржаной хлеб 400г — только мука, вода, соль. Без дрожжей» (акцент: состав)
Гипотеза Б: «Хлеб ржаной 400г — 89₽. Свежая выпечка каждый день» (акцент: цена)

Этап 3 · Размещение
Публикация объявлений
Два разных пути для услуг (через API) и товаров (через XML)
▼
Суть этапа
Готовые варианты объявлений нужно опубликовать на Авито. Здесь процесс разделяется на два пути в зависимости от типа: услуги публикуются через API Авито (прямое подключение через интернет), а товары — через XML-выгрузку (файл с данными, который загружается на FTP-сервер Авито).

Для каждого города создаётся отдельное объявление. После публикации Авито возвращает уникальный номер объявления (avito_ad_id) — его сохраняем в своей базе данных. Это нужно для дальнейшего сбора статистики.
Два пути публикации
🛎️ Услуга: «Испечение хлеба»
  • Python-скрипт отправляет POST-запрос к API Авито
  • Передаёт: текст, картинку (URL на хостинге), ставку за показ
  • Авито отвечает: avito_ad_id: 3847562
  • Сохраняем ID в базу данных
  • Для Москвы — одно объявление, для Питера — другое
🛒 Товар: «Хлеб и пироги»
  • Скрипт формирует XML-файл со всеми SKU: тексты, картинки, города, уникальные поля, ставки
  • Файл загружается на FTP-сервер Авито
  • Авито обрабатывает файл раз в час
  • После обработки получаем avito_ad_id для каждого SKU
  • Сохраняем все ID в базу данных
Поток данных
📄 variants.csv → 🔌 API Авито → 💾 avito_ad_id в БД (услуги)
📄 variants.csv → 📄 XML → FTP Авито → 💾 avito_ad_id в БД (товары)
Важно знать (для отдела продаж)
ℹ️API — это прямое автоматическое подключение к Авито. Не нужно вручную заходить в кабинет и нажимать кнопки — скрипт делает всё сам.
ℹ️XML-выгрузка — стандартный способ для магазинов. Файл описывает все товары, и Авито сам их обрабатывает. Обновление происходит автоматически раз в час.
⚠️Авито иногда блокирует объявления — даже без нарушений. На этот случай есть автоматическое пересоздание (см. Этап 4).

Этап 4 · Мониторинг
Сбор статистики
Ежедневно собираем данные по каждому объявлению: показы, клики, контакты
▼
Суть этапа
Каждый день запускается скрипт, который проходит по всем объявлениям в нашей базе и запрашивает у Авито статистику: сколько раз показали объявление, сколько человек кликнули, сколько написали или позвонили, сколько денег потрачено.

Данные сохраняются в базу данных (SQLite — это файл на компьютере, работающий как таблица). Каждая строчка — это один день, одно объявление. Накопленная история позволяет увидеть динамику и сравнивать варианты.

Если объявление заблокировано Авито — это фиксируется. Автоматически создаётся новое объявление для того же товара, но с изменёнными уникальными полями (другой артикул, другой вес). Новый avito_ad_id привязывается к тому же внутреннему ID — статистика не теряется.
Кто что делает
👤 Специалист
  • Настраивает расписание запуска скрипта (раз в день или чаще)
  • Контролирует, что сбор проходит без ошибок
  • При необходимости — вручную проверяет подозрительные блокировки
🤖 Python + API Авито
  • Автоматически обходит все avito_ad_id из базы
  • Запрашивает у Авито статистику за последние сутки
  • Записывает данные в SQLite с датой и ID объявления
  • Проверяет статус каждого объявления — не заблокировано ли
  • При блокировке — создаёт новое объявление автоматически
Поток данных
💾 avito_ad_id из БД → 🔌 API Авито (статистика) → 💾 SQLite: показы, клики, контакты, затраты по дням
❌ Заблокированное объявление → 🔄 Скрипт пересоздания → ✅ Новый avito_ad_id, статистика сохранена
Что собирается
👁️Показы — сколько раз объявление появилось перед пользователями Авито
🖱️Клики — сколько человек перешли на объявление
📞Контакты — сколько человек написали или попросили показать номер телефона
💸Затраты — сколько денег списалось за день по этому объявлению

Этап 5 · Анализ
Анализ гипотез
Сравниваем варианты объявлений по CTR, цене контакта и CPA
▼
Суть этапа
Накопленная статистика агрегируется по гипотезам (вариантам объявлений). Если одна гипотеза была и заблокирована, и пересоздана — показатели нового и старого объявления складываются, чтобы видеть полную картину.

Для каждой гипотезы считаются три ключевые метрики: CTR (интерес аудитории), цена контакта (эффективность бюджета) и CPA (стоимость сделки, если есть данные из CRM). По ним можно быстро понять, какой вариант работает лучше.
Ключевые метрики
CTR = клики / показы × 100%
Кликабельность — насколько объявление привлекательно
CPL = затраты / контакты
Цена за контакт — во сколько обходится один звонок/сообщение
CPA = затраты / продажи
Стоимость сделки — если подключена CRM с данными о продажах
Пример анализа
Услуга «Испечение хлеба» — результаты за 7 дней:

Гипотеза А «Свежий хлеб за 3 часа»: CTR 1.8%, контакт 43₽ → ЛИДЕР 🏆
Гипотеза Б «Домашняя выпечка»: CTR 0.4%, контакт 210₽ → Отключить ❌

Вывод: Акцент на скорость и конкретную пользу работает лучше, чем общие слова про «домашнее».
Поток данных
💾 SQLite статистика → 🐼 pandas / Python → 📊 Таблица CTR, CPL, CPA по каждой гипотезе

Этап 6 · Оптимизация
Управление ставками
Поднимаем ставки у эффективных объявлений, снижаем у слабых
▼
Суть этапа
На Авито объявления конкурируют за позиции — чем выше ставка (бюджет в день), тем чаще и выше показывается объявление. Задача — найти баланс: не тратить деньги на слабые объявления и усилить показ тех, что приносят дешёвые контакты.

Правило простое: если объявление кликают и звонят — повышаем ставку, чтобы его видело больше людей. Если не кликают — снижаем или останавливаем, пока не поймём причину или не сделаем новую гипотезу.
Правила принятия решений
📈CTR > 1% + цена контакта низкая: повысить ставку на 10–20%. Объявление работает — даём ему больше показов.
📉CTR < 0.5% + потрачено больше лимита: снизить ставку или поставить на паузу. Не работает — не тратим деньги.
🔄Гипотеза провальная: заменить на новую. Возвращаемся к Этапу 2, делаем другой текст/картинку.
Как технически меняются ставки
🛎️ Услуги (API)
  • Запускается Python-скрипт
  • Скрипт отправляет команду в API Авито: «для объявления ID xxx установить ставку Y рублей»
  • Авито применяет изменения мгновенно
🛒 Товары (XML)
  • В XML-файле меняется значение ставки для нужных позиций
  • Обновлённый файл загружается на FTP Авито
  • Авито применяет при следующей обработке файла (до 1 часа)
Поток данных
📊 Таблица метрик → 🔌 API Авито / FTP → ✅ Обновлённые ставки в объявлениях

Этап 7 · Рост
Масштабирование
Победившую гипотезу применяем ко всем похожим SKU
▼
Суть этапа
Когда нашли формулу, которая работает — нет смысла ограничиваться одним товаром. Если текст «Свежий хлеб за 3 часа» отлично конвертирует — значит, похожий подход (скорость + конкретная польза) скорее всего сработает и для пирогов, булочек, других позиций из той же категории.

Берём победивший шаблон и применяем его ко всем SKU, где эта гипотеза ещё не тестировалась. Скрипт создаёт новые объявления с тем же структурным подходом, но уникальными полями и адаптированным текстом для каждой позиции.
Пример масштабирования
Победила гипотеза: акцент на скорость и натуральность для «Хлеба ржаного 400г»

Применяем к:
• «Булочки с корицей 300г» → «Булочки с корицей — свежие за 2 часа, без красителей»
• «Пирог с яблоком 500г» → «Пирог с яблоком — домашний рецепт, готов к вечеру»
• «Хлеб пшеничный 400г» → «Пшеничный хлеб без дрожжей — выпекаем каждый день»

Все новые объявления добавляются в систему сбора статистики.
Поток данных
🏆 Лучшая гипотеза + 📄 Остаток SKU из catalog.csv → 🤖 Claude API + Python → 📈 Новые объявления → больше заявок

Этап 8 · Цикл
Отчётность и итерации
Регулярные отчёты и постоянное улучшение показателей
▼
Суть этапа
В конце каждой недели и месяца собирается отчёт: как менялись показатели, какие тексты и картинки сработали, сколько потрачено, сколько получено контактов, какая стоимость заявки.

На основе отчёта принимаются решения на следующий цикл: что отключить, что усилить, какие новые гипотезы протестировать. Процесс не заканчивается — он постоянно улучшается. Каждый цикл даёт новые данные, которые делают следующий цикл эффективнее.
Что входит в отчёт
📈Динамика по кампаниям: как менялись показы, клики, контакты по дням и неделям
🏆Топ гипотез: какие тексты и картинки сработали лучше всего и почему
💸Финансы: общие затраты, количество контактов, средняя цена контакта, CPA
📋План на следующий период: новые гипотезы, изменения ставок, масштабирование
Поток данных
💾 SQLite (все накопленные данные) → 🐼 pandas / Python → 📋 Отчёт (неделя / месяц)
📋 Отчёт → 🔁 Следующий цикл (→ Этап 2)

Вопросы и ответы

Часто спрашивают

Ответы на типичные вопросы — удобно для отдела продаж при работе с клиентами.

Сколько времени занимает запуск? ▼
От получения прайс-листа до первых опубликованных объявлений — обычно 2–3 рабочих дня. Это включает подготовку данных, генерацию текстов и картинок, тестовую публикацию. Первая статистика появится через 3–7 дней после публикации.
Что такое «гипотеза» и зачем их несколько? ▼
Гипотеза — это один вариант подачи объявления: определённый заголовок, описание, картинка. Заранее неизвестно, что лучше сработает на вашу аудиторию. Поэтому делаем 2–4 варианта, публикуем, смотрим на статистику и оставляем лучший. Это называется A/B-тестирование.
Почему объявления иногда блокирует Авито? ▼
Авито автоматически проверяет объявления на дубли и нарушения. Иногда алгоритм срабатывает ошибочно — даже у честных объявлений. Именно поэтому у каждого объявления генерируются уникальные поля (артикул, вес, размер) — это снижает вероятность блокировки. А если блокировка всё же случилась — система автоматически создаёт новое объявление.
Нужен ли у клиента аккаунт на Авито? ▼
Да, нужен аккаунт Авито с подключённым тарифом, который разрешает работу через API или XML-выгрузку. Обычно это тарифы для бизнеса. Помогаем с настройкой и подключением.
Как считается цена контакта? ▼
Цена контакта (CPL) = все затраты на объявление за период ÷ количество контактов (звонки + сообщения) за тот же период. Например: потратили 2 000₽, получили 20 контактов — цена контакта 100₽. Наша задача её снижать.
Это ручная работа или автоматическая? ▼
Сейчас процесс полуавтоматический: ключевые операции (сбор статистики, генерация текстов, публикация) автоматизированы через скрипты, но специалист контролирует результаты, принимает решения по оптимизации и формирует отчёты. Постепенно всё больше операций переходит в полностью автоматический режим.
Что значит «масштабирование»? ▼
Когда мы нашли работающий подход (гипотезу с хорошим CTR и низкой ценой контакта), мы не ограничиваемся одним товаром. Берём тот же успешный шаблон и применяем его ко всем похожим позициям в каталоге. Это позволяет быстро увеличить количество заявок без долгого тестирования с нуля.
